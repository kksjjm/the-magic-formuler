# 산식의 마법사 개발 정리

작성일: 2026-06-07

## 1. 게임 설명

『산식의 마법사 (The Equation Mage)』는 "마법은 곧 수식이다"라는 콘셉트의 웹 기반 제작·실험 샌드박스 게임이다. 현재 버전은 플레이어가 블록을 고르는 방식이 아니라, 전용 언어인 MagicScript로 마법 코드를 작성하면 바로 옆 실험장에서 컴파일 결과와 실행 효과를 시각적으로 확인하는 코드형 마법 실험실이다.

플레이어는 `range`, `target`, `set`, `push`, `swirl`, `output` 같은 줄 단위 명령으로 물질의 운동량, 온도, 전하, 응집도를 바꾼다. 시뮬레이터는 이 변화를 물, 공기, 흙의 상태 전이 규칙에 반영해 얼음, 증기, 플라즈마, 유리 같은 현상을 보여준다.

## 3. 각 파일의 역할

| 파일 | 역할 |
|---|---|
| `index.html` | 게임의 단일 진입점. MagicScript 코드 편집기, 실행 프리뷰 캔버스, 실행 분석, 저장/공유 UI를 배치한다. |
| `src/styles.css` | 코드 편집기 중심 레이아웃, 캔버스 실험장, 분석 패널, 저장 목록, 반응형 화면 스타일을 담당한다. |
| `src/main.js` | 월드 생성, 실시간 시뮬레이션, MagicScript 컴파일 결과 실행, 비용 계산, 챌린지 판정, 저장/공유 기능을 담당한다. |
| `src/magic-script.js` | MagicScript v0 파서, 예시 마법 코드, 컴파일 요약 생성 로직을 담당한다. |
| `src/data/substances.json` | 물, 공기, 흙의 속성, 상태 전이 규칙, 마나 가중치, 응집 규칙을 정의한다. |
| `src/data/challenges.json` | 자유 실험과 선택 도전 과제의 목표, 메달 기준을 정의한다. |
| `src/data/blocks.json` | 이전 블록형 MVP의 블록 데이터다. 현재 UI에서는 직접 사용하지 않지만, 향후 블록-코드 변환이나 하위 호환에 활용 가능하다. |
| `src/data/presets.json` | 이전 블록형 MVP의 예시 레시피 데이터다. 현재 예시는 `src/magic-script.js`의 코드 예시로 이동했다. |
| `MagicScript_언어명세.md` | 전용 마법 프로그래밍 언어의 현재 문법과 제한 사항을 정리한 문서다. |
| `산식의_마법사_게임기획서.md` | 원본 게임 디자인 문서다. 구현 방향과 장기 로드맵의 기준 문서로 보존한다. |
| `개발_정리.md` | 매 작업 후 게임 설명, 구조, 진행 내용, 다음 개발 항목을 갱신하는 문서다. |

## 4. 지금까지 진행된 개발내용

- 기존 블록 선택형 마법 편집기를 코드 작성형 MagicScript 편집기로 전환했다.
- MagicScript v0 언어 체계를 구현했다.
  - `spell`
  - `range circle`, `range rect`, `range lane`
  - `target`
  - `set`
  - `push aim`
  - `swirl`
  - `output release`, `focus`, `bloom`, `anchor`
- 브라우저에서 JavaScript를 직접 실행하지 않고 허용된 명령만 파싱하는 안전한 파서 구조를 추가했다.
- 코드 변경 시 컴파일 상태, 대상 수, 범위 오버레이, 비용 패널이 즉시 갱신되도록 구현했다.
- 자동 실행 토글을 추가해 코드를 수정하면 프리뷰 월드에서 바로 실행 효과를 확인할 수 있게 했다.
- 아이스볼, 증기분수, 정전기 고리, 유리 씨앗을 MagicScript 예시 코드로 제공한다.
- 마법 이름 입력, 코드 저장, 저장된 마법 열기, 삭제 기능을 코드 기반으로 변경했다.
- 공유 코드 내보내기/불러오기를 MagicScript 소스 기반으로 변경했다.
- 기존 캔버스 시뮬레이션, 물질 상태 전이, 마나/시전 시간 계산, 챌린지 UI를 새 코드 실행 흐름에 연결했다.
- 검증 결과:
  - `src/main.js` ESM 문법 검사 통과
  - `src/magic-script.js` ESM 문법 검사 통과
  - `src/data/substances.json`, `src/data/challenges.json` 파싱 통과
  - 인앱 브라우저에서 새 UI 로드 확인
  - 예시 코드 전환, 자동 프리뷰, 저장, 공유 코드 생성 확인
  - 브라우저 콘솔 오류 없음

## 5. 앞으로 개발해야할 내용

- MagicScript v1 문법 확장:
  - `let` 변수
  - 거리 기반 값
  - 조건 분기
  - 안전한 수식 표현식
  - 사용자 정의 함수 또는 매크로
- 코드 에디터 품질 개선:
  - 줄 번호
  - 구문 강조
  - 오류 라인 하이라이트
  - 자동 완성
- 저장된 마법에 설명, 태그, 썸네일, 실행 결과 스냅샷을 붙일 수 있게 한다.
- 공유 코드를 더 짧게 압축하고, 불러오기 전에 이름과 미리보기를 보여준다.
- 코드 실행 리플레이와 단계별 디버거를 추가한다.
- `blocks.json` 기반 블록 UI와 MagicScript를 서로 변환하는 하이브리드 편집기를 검토한다.
- Tier 2 준-물리 입자 시뮬레이션을 별도 엔진 모듈로 분리한다.
- Tier 3의 직접 수식 입력을 MagicScript의 안전 DSL로 통합한다.
