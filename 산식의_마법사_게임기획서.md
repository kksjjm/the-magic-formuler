> "외우지 마라. 조합하라. 그리고 무슨 일이 벌어지는지 지켜봐라."

## 2. 핵심 재미 (Design Pillars) — 샌드박스 우선순위

1. **발견 · 창발 (Discovery & Emergence) — 최우선**
   속성(운동량·온도 등)을 조작하면 응집·상전이·연쇄반응 같은 거시 현상이 *예측 못 한 방식으로* 나타난다. "이렇게 했더니 이런 일이?"의 쾌감.

2. **자유로운 제작 (Toys, not Tasks)**
   정답이 정해진 퍼즐을 푸는 게 아니라, 블록을 마음대로 조합해 *나만의 마법*을 만든다. 실패도 결과물이고, 쓸모없는 마법도 만들 자유가 있다.

3. **선택적 최적화 깊이 (Optional Mastery)**
   원하는 사람만 파고드는 층위. 연산속도·마나량을 줄이는 *효율 경쟁*은 강제가 아니라 자발적 도전·랭킹으로 제공.

4. **공유와 과시 (Share & Show off)**
   만든 마법을 저장·공유하고, 남의 마법을 가져와 개조한다. 샌드박스의 수명을 커뮤니티가 연장한다.

> **샌드박스 원칙:** 시스템은 "벽"이 아니라 "재료"다. 비용·제약은 막는 장치가 아니라 *피드백과 선택적 도전*으로 제시한다.

## 3. 핵심 게임 루프 (샌드박스 중심)

```
        ┌────────── [자유 실험: 핵심] ──────────┐
        │  마법 편집기에서 블록 조합 → 실행      │
        │  → 창발 현상 관찰 → "오 이거 되네?"   │
        │  → 다시 만지작 (즉각 피드백 루프)      │
        └───────────────┬───────────────────────┘
                        │ (원하면)
     ┌──────────────────┼──────────────────────┐
     ▼                  ▼                       ▼
[선택 도전]        [공유/개조]              [내러티브 캠페인]
효율·속도 챌린지    내 마법 저장·공개         괴짜 마법사와 함께
스스로/랭킹 경쟁    남의 마법 리믹스          차원이동 마법 완성(가벼운 가이드)
     │                  │                       │
     └──────────────────┴───────────────────────┘
                        ▼
        해금: 새 블록·속성·연산자·물질 → 더 넓은 실험 공간
```

- **핵심 루프(초 단위):** 조합 → 실행 → 관찰 → 수정. *진입장벽 낮고 즉각적인* 피드백이 생명.
- **확장 루프:** 선택적 도전 / 공유·리믹스 / 느슨한 서사 캠페인 — 셋 다 *옵션*.
- **해금:** 캠페인 진행, 도전 클리어, 혹은 단순 탐험으로 새 재료를 얻어 실험 공간을 넓힌다.

> 낮/밤 경영 루프(마법서 판매)는 **강제 메인 루프에서 제외**하고, 원하는 사람을 위한 *선택적 "마법상점 모드"*로 격하한다 (6장).

## 4. 마법 시스템 (핵심 메커니즘)

### 4.1 마법의 4단계 파이프라인

| 단계 | 역할 | 비용 신호(피드백) |
|---|---|---|
| ① 범위지정 (Range) | 작용할 공간 영역 정의 | 스캔 연산속도 (부피·해상도 비례) |
| ② 객체인식 (Recognition) | 범위 내 작용 대상 필터링 | 필터 연산속도 (객체수×조건수) |
| ③ 산식적용 (Apply) | 대상의 **속성값을 함수로 변환** (마나 소모 핵심) | 마나량 (대상수×변화량), 산식 복잡도→연산속도 |
| ④ 결과출력 (Output) | 변환된 속성이 시뮬레이션에 반영되어 현상 발생 | — |

> 샌드박스에서 ②↔③ 트레이드오프(필터를 촘촘히 = 느리지만 마나 절약 / 무차별 = 빠르지만 마나 폭증)는 **막는 규칙이 아니라 "더 잘 만들고 싶은 사람"을 위한 깊이**로 제공한다.

### 4.2 월드 데이터 모델 (데이터 주도 설계)

모든 물질·규칙·연산자·블록을 **외부 데이터(JSON/스크립트)로 정의** → 코드 수정 없이 콘텐츠 확장(비개발자·AI가 데이터만 추가). 웹/모바일 패치도 데이터 갱신으로 처리.

**객체(Object) 예시**
```json
{
  "id": 1042, "type": "H2O",
  "position": [x,y,z], "velocity": [vx,vy,vz], "mass": 1.0,
  "momentum": 0.8, "acceleration": [ax,ay,az],
  "temperature": 293, "charge": 0, "state": "liquid"
}
```

**물질 타입(Substance) — 상태 전이 규칙**
```json
{
  "type": "H2O",
  "attributes": ["momentum","temperature","charge"],
  "transitions": [
    { "if": "momentum < 0.2 && temperature < 273", "state": "ice",   "visual": "ice_crystal" },
    { "if": "momentum >= 0.2",                      "state": "water", "visual": "water" },
    { "if": "temperature > 373",                    "state": "vapor", "visual": "steam" }
  ],
  "aggregation_rule": "운동량 낮은 인접 객체끼리 응집"
}
```

→ ③에서 운동량을 낮추면 엔진이 규칙을 평가해 "얼음"이라는 *창발*을 자동 생성. 이 규칙 테이블 확장이 콘텐츠 제작의 핵심.

### 4.3 비용 모델 (정량 공식 — 피드백·랭킹의 기반)

샌드박스에선 자원이 "벽"이 아니라 **점수·피드백**이지만, 도전/랭킹/밸런싱을 위해 정량 공식이 필요하다.

**(A) 마나량**
```
ManaCost = Σ(대상 객체) Σ(변경 속성)  |Δ속성| × W_attr × D_substance
```

**(B) 연산속도 = 캐스팅 시간**
```
CastTime = T_base
         + (범위 부피 / 스캔 해상도) × C_scan
         + (범위 내 객체수 × 필터 조건수) × C_filter
         + 산식복잡도(연산자 종류·중첩) × C_apply
```

**(C) 트레이드오프 시연 — "아이스볼"** (분자 1,000개 중 물 300개)

| 전략 | ② 필터 | 캐스팅 | ③ 대상 | 마나 | 성격 |
|---|---|---|---|---|---|
| A. 물 분자만 | `type==H2O` | 느림 | 300 | 적음 | 효율형 |
| B. 무차별 | 없음 | 빠름 | 1000 | 많음 | 즉발형 |
| C. 좁은 범위+필터 | 좁은 구+`type==H2O` | 중간 | 80 | 최소 | 사거리 희생, 최효율 |

→ 상황에 따라 최적해가 달라짐. 샌드박스에선 "정답 강요" 대신 *효율 점수·메달*로 자발적 도전 유도.

**(D) 효과 강도 ↔ 마나**
```
사거리 ∝ 가속도값 ;  도달시간 ∝ 1/가속도값 ;  ManaCost ∝ |Δ가속도| × 대상수
```

### 4.4 마법 편집기 (블록 조합 UI · 터치/마우스 겸용)

- **블록 팔레트:** 범위 / 인식(필터) / 산식(연산자) / 출력 블록을 드래그·탭으로 조립.
- **산식 라이브러리 = 미리 만들어진 식들(1단계) → 점차 직접 작성(3단계).**
  - 기본: 상수 가감, 곱셈, set(고정값)
  - 중급: 거리 비례 그라디언트, 조건 분기
  - 고급(해금): 역제곱 `F=G·M/r²`, 지수 감쇠, 벡터 회전 등 — 강력하지만 `C_apply`↑
- **실시간 비용 패널:** 조작 즉시 마나·캐스팅·등급 갱신 → 빠른 실험 루프.
- **시뮬레이터/디버거:** 슬로우·스텝 재생, 객체별 속성 변화 오버레이.
- **모바일 고려:** 큰 탭 타깃, 스냅 연결, 핀치 줌, 핵심 동작은 한 손 조작 가능.

### 4.5 제작 워크스루 예시 (개발자 참고)

"아이스볼": ①전방 반경 3m 구 → ②`type==H2O` → ③`momentum-=0.7`(얼음 창발) → ②`state==ice` → ③`accel.dir=조준점, accel.value+=K` → ④발사.

### 4.6 시뮬레이션 진화 로드맵 (★ 개발 단계 = 플레이 심화 단계)

확정된 단계적 심화 전략. **각 단계가 곧 개발 페이즈이자 유저 숙련 램프**다.

| 단계 | 시뮬 방식 | 플레이 경험 | 기술/성능 | 개발 페이즈 |
|---|---|---|---|---|
| **Tier 1. 추상 규칙 기반** | 이미 만들어진 산식 블록을 *조합*. 객체를 추상 집합("물 무리")으로 다루고 상태 전이를 규칙 테이블로 즉시 판정. 개별 입자 시뮬 X | 진입장벽 최저. "레고처럼" 조합·관찰 | 매우 가벼움 → **모바일 웹에서도 쾌적** | **MVP / 출시 기반** |
| **Tier 2. 준-물리 입자 시뮬** | 실제 입자들이 속도·충돌·응집을 시뮬. 창발이 진짜 물리에서 발생 | 더 깊은 실험·예측 불가 현상 | WebGPU/WASM로 성능 확보, 입자수 상한·LOD, 저사양은 Tier1 폴백 | 2차 업데이트 |
| **Tier 3. 수식 입력형** | 플레이어가 함수/수식을 *직접 작성*(비주얼 수식 또는 안전 DSL) | 코어·교육 유저의 정점, 무한 확장 | 샌드박스 평가기 필수(무한루프·악성 차단, 실행시간 제한) | 3차/장기 |

> 설계 일관성: 세 단계 모두 **4.1의 4단계 파이프라인을 공유**한다. 달라지는 건 ③ 산식이 "고른다 → 직접 쓴다"로 진화하고, 시뮬 백엔드가 "규칙판정 → 입자물리"로 깊어지는 것뿐. 따라서 비용 엔진(4.3)·데이터 모델(4.2)은 처음부터 3단계를 염두에 두고 설계해야 함.

## 5. 진행 / 성장 (해금) 시스템

- **해금 = 강함이 아니라 "넓어짐".** 새 속성·연산자·물질·상태 규칙이 풀리면 실험 가능 공간이 확장된다.
- **해금 경로(전부 선택적):** 느슨한 서사 캠페인 진행 / 선택 도전 클리어 / 자유 탐험 중 우연한 발견.
- **장비·환경 강화:** 마나풀 용량, 연산 코어(캐스팅↓), 스캔 해상도, 더 큰 실험장 등.

## 6. 선택적 "마법상점 모드" (구 경영 루프)

원작 줄거리의 "낮 판매 / 밤 연구"는 **메인이 아닌 선택 모드**로 보존:
- 손님이 효과·제약(마나·시간)을 의뢰 → 만든 마법서로 충족 시 보상·명성.
- 샌드박스 본편과 분리된 *목표 지향 모드*를 원하는 유저용. 서사 캠페인의 한 챕터로도 활용 가능.

---

## 7. 선택적 도전 / 시나리오

- **형태:** 강제 전투 대신 "샌드박스 챌린지"(예: "물 없는 사막에서 얼음 만들기", "마나 50 이하로 X 달성", "2초 내 시전").
- **평가:** 목표 달성 + 마나 + 시간 + 부수효과 → 메달/리더보드(공유 연계).
- 전투형 콘텐츠는 후순위 확장(13장 5번).

---

## 8. 내러티브 (가벼운 캠페인 = 튜토리얼+세계관)

미래 지구 최고 과학자가 블랙홀 사고로 '마나의 세계'에 추락 → 괴짜 마법사의 마법진 위로 떨어져 사제·협력 관계 형성 → 그가 만들던 *차원이동 마법*을 함께 완성.

샌드박스 중심이므로 서사는 **강제 진행 대신 느슨한 가이드 캠페인**으로:
1. 추락/마법진 복원 — 4단계 파이프라인 튜토리얼
2. 첫 마법(아이스볼) — 인식·산식·창발 체험
3~5. 마법사의 의뢰를 따라가며 새 재료 해금(샌드박스 자유 실험은 항상 병행 가능)
6. 차원이동 마법진 완성 → 출발 **(행선지 미정, 13장 6번)**

테마: "미지를 외우지 않고 *이해하여* 정복한다."

## 9. UI / UX (반응형 · 터치/마우스)

- **화면:** ① 샌드박스 실험장(메인) ② 마법 편집기 ③ 해금/지식 ④ 공유 갤러리 ⑤ (선택)상점·캠페인.
- **편집기 레이아웃:** 데스크톱=좌 팔레트/중앙 캔버스/우 비용패널, 모바일=하단 시트형 팔레트+전체화면 캔버스.
- **블록 표기:** 자연어+수식 병기 (예: `운동량 −0.7  (momentum -= 0.7)`).
- **반응형:** 동일 코드, 브레이크포인트로 레이아웃 전환. 터치 제스처(드래그/핀치/롱프레스) 1급 지원.

## 10. 아트 / 사운드 방향 (미정 — 13장)

- 후보 A: 미니멀 다이어그램·홀로그래픽 SF(청사진·수식 감성) — 샌드박스 가독성에 유리
- 후보 B: 도트/2D 일러스트(따뜻한 인디)
- 사운드: 시전 "연산 처리음" + 창발 현상(결빙·폭발)의 물리 효과음 대비. 웹 오디오 경량화.

## 11. 기술 스택 / 개발 고려사항 (웹 · PC+모바일)

**클라이언트(웹)**
- 언어: **TypeScript**.
- 렌더링/게임층 후보: PixiJS(2D, 경량) 또는 Phaser, 혹은 Godot/Unity의 WebGL 익스포트. *Tier1은 가벼운 2D 권장.*
- 입자/물리(Tier2): **WebGPU**(우선) 또는 WebGL2 컴퓨트, 무거운 연산은 **WASM**(Rust/C++)로. 미지원 기기는 WebGL/CPU 폴백.
- 수식 평가(Tier3): **샌드박스 평가기**(전용 DSL 또는 제한된 표현식 파서). 실행시간·메모리·루프 제한 필수(보안·안정성).

**아키텍처**
- 객체를 **SoA 배열 / ECS**로 관리(수천 객체 성능). 마법 실행 = "객체 배열에 대한 필터+변환 파이프라인"(함수형).
- 상태 전이는 매 틱 규칙 평가, **공간 그리드/쿼드트리**로 인접 응집 가속.
- **결정적 시뮬레이션**(고정 시드·고정 틱) → 리플레이·공유 검증·디버깅 가능. *공유 기능의 전제 조건.*
- **비용 산출 엔진 단일화**: 4.3 공식을 한 모듈로 → 편집기·평가·랭킹·밸런싱이 같은 소스 공유.

**플랫폼/성능 예산**
- 모바일 웹 60fps 목표 → Tier1 객체수 상한, LOD, 오프스크린 컬링.
- 단일 코드베이스 + 반응형. 추후 Capacitor/PWA로 앱 래핑 가능.

**데이터 주도**
- 물질·전이규칙·연산자·블록·의뢰·챌린지를 모두 JSON으로 외부화 → 콘텐츠/밸런스 핫패치, AI 생성 콘텐츠 투입 용이.

## 12. MVP 범위 (= Tier 1 추상 규칙 샌드박스)

**목표:** "조합해서 창발을 발견하는 재미"가 웹+모바일에서 작동하는지 검증.

포함:
- 4단계 파이프라인 편집기(블록 6~10종), 터치/마우스 지원, 반응형
- 추상 규칙 시뮬(Tier1) + 물질 2~3종(물·공기·흙) + 상태 전이 규칙
- 실시간 비용 패널(마나·캐스팅) + 슬로우/스텝 시뮬레이터
- 마법 저장 + 간단 공유(링크/코드로 레시피 내보내기·불러오기)
- 예시 마법 2~3종(아이스볼 등) + 선택 챌린지 3~5개
- 가벼운 서사 인트로(튜토리얼 1~2챕터)

제외(후순위): 입자물리(Tier2)·수식입력(Tier3), 상점 경영 모드, 전투, 풀 캠페인, 본격 커뮤니티 갤러리.

**성공 기준:** 테스터가 "주어진 목표 없이도" 이것저것 조합하며 *스스로 무언가를 만들어내고 공유하고 싶어 하는가.*
